# 阶段门禁

## 通用门禁

每个阶段必须同时满足：

1. 自动测试、格式检查和敏感内容检查通过；
2. 本阶段交付物可由 README 或当前状态文档定位；
3. 私有资料、密钥和运行产物未进入 Git；
4. 验收结果记录在阶段文件与变更日志中；
5. 下一阶段的阻塞输入已经登记。

## 阶段门禁表

| 阶段 | 进入条件 | 必交付物 | 自动检查 | 人工验收 | 失败回退 |
|---|---|---|---|---|---|
| S0 | 设计规格批准 | 项目管理与隐私基线 | Hook 与文档契约测试 | 范围审阅 | 修订设计或决策日志 |
| S1 | S0 完成 | 合成纵向切片 | Ruff、pytest、双平台 CI | CLI/API 演示 | 保留合成数据，修复接口 |
| S2 | 一病种资料获准使用 | 真实资料导入与生成链路 | 导入、检索、契约测试 | 医学有效性审阅 | 回退合成语料或更换资料 |
| S3 | 教学目标与示范问答到位 | 问题答案包 | 锚点与 Schema 测试 | 逐题审核 | 返回 Prompt/证据设计 |
| S4 | 质控规则和错误样例到位 | 三轮质控闭环 | 回归集指标 | 医学专家复核 | 降级为人工审核 |
| S5 | 审核人和量表确认 | 发布状态机 | 状态转换测试 | 发布演练 | 撤回并重新审核 |
| S6 | 后端主流程稳定 | Web MVP | 浏览器验收测试 | 医学用户体验 | 回退 API/CLI 继续使用 |
| S7 | 六病种资料到位 | 30 个发布病例 | 数量和状态审计 | 逐例签署 | 未通过病例退回草稿 |
| S8 | 伦理和研究方案批准 | 冻结评价数据 | 数据完整性检查 | 统计方案复核 | 不进入正式教学研究 |
| S9 | 正式分析完成 | 结题成果材料 | 文档和版本审计 | 项目组验收 | 修订材料 |
