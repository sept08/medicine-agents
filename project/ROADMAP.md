# 项目路线图

| 阶段 | 可体验结果 | 工程输出 | 医学团队输入 | 退出条件 |
|---|---|---|---|---|
| S0 项目基线 | 可导航的项目管理体系 | 范围、路线图、门禁、隐私基线 | 确认 MVP 病种 | 基线文档通过审阅 |
| S1 最小骨架 | 合成订单生成固定病例包 | Schema、文件仓库、CLI、API | 无 | 离线全链路与测试通过 |
| S2 单病种生成 | 一个病种基于资料生成病例 | 导入、检索、商业模型适配器 | 教材/指南、至少 3 份脱敏病例 | 医学人员确认可用性 |
| S3 问题答案包 | 阶梯问题及有依据的答案 | 出题工作流、证据绑定 | 教学目标、5–10 组示范问答 | 逐题答案验收通过 |
| S4 质控闭环 | 最多三轮自动修正 | 规则、模型质控、质控报告 | 医学规则、错误样例 | 目标测试集达到门槛 |
| S5 发布治理 | 编辑、审核、发布和归档 | 状态机、版本、审核记录 | 审核人、最小量表 | 未审核内容不可发布 |
| S6 Web MVP | 浏览器跑通完整流程 | React 界面与后端集成 | 可用性反馈 | 主流程验收通过 |
| S7 六病种 MVP | 30 个正式病例包 | 六病种本地病例库 | 六病种资料及逐例审核 | 30 例均审核发布 |
| S8 正式评价 | 冻结 100 例评价集 | 评价流水线和分析数据 | 专家、伦理、量表、教学安排 | 质量与研究数据完整 |
| S9 成果整理 | 可用于结题和成果申报 | 技术文档、报告和材料 | 最终统计与医学审阅 | 成果材料通过项目组验收 |

## 并行原则

医学资料准备与工程开发并行。只有缺少某项输入会使当前阶段无法达到退出条件时，才将其升级为阻塞项。
